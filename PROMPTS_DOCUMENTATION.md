# Paper Interpreter - Prompt 文档

本文档记录了系统中所有用于生成科普文章的 Prompt。

---

## 1. 论文分析 Prompt (Analyzer)

**位置**: `paper_to_popsci/core/analyzer.py` - `_build_analysis_prompt()`

**作用**: 分析论文内容,生成文章大纲和结构

**Prompt 内容**:

```
你是一位资深科普作家。请分析以下学术论文，提取关键信息并规划一篇面向普通读者的科普文章结构。

# 论文信息

标题: {title}
摘要: {abstract}
主要章节: {sections_text}

# 任务要求

请仔细阅读论文内容，然后输出 JSON 格式的分析结果。

## 重要提示:
1. **标题必须吸引人**: 不要直接用论文标题，要转化成普通人能理解且感兴趣的科普标题
2. **类比要具体**: 选择一个贴近生活的场景，能让完全不懂技术的人也能理解
3. **痛点要清晰**: 用大白话说明现有方法的问题
4. **关键概念要提炼**: 从论文中提取3个最核心的技术点

## JSON 格式:
{
    "article_type": "论文类型(从以下选择: 架构创新/算法优化/应用突破/理论分析)",
    "core_innovation": "用一句通俗的话概括核心创新点(不超过30字)",
    "analogy_theme": "选择一个具体的日常生活场景(如: 餐厅点餐/图书馆借书/快递分拣/城市交通调度)",
    "sections": [
        {
            "type": "hero",
            "title": "科普标题(必须吸引人、通俗易懂，让人一看就想读，15-25字)",
            "subtitle": "副标题(用引号包裹，一句话说明这项技术的价值，10-20字)"
        },
        {
            "type": "intro",
            "title": "引子标题(格式: 从XX谈起，如'从外卖配送谈起')",
            "analogy": "具体的生活场景描述(50-100字，要生动形象)"
        },
        {
            "type": "problem",
            "title": "问题标题(如: 传统方法的困境)",
            "pain_point": "用大白话描述现有方法的核心问题(30-50字)"
        },
        {
            "type": "method",
            "title": "方法标题(如: 创新的解决方案)",
            "key_concepts": ["从论文中提取的关键技术点1", "关键技术点2", "关键技术点3"]
        },
        {
            "type": "results",
            "title": "结果标题(如: 效果如何？数字说话)",
            "metrics": [
                {"name": "性能指标名称", "value": "具体数值", "meaning": "这个数字意味着什么"}
            ]
        },
        {
            "type": "impact",
            "title": "意义标题(如: 这对我们有什么影响？)",
            "implications": ["对普通人生活的具体影响1", "具体影响2"]
        },
        {
            "type": "conclusion",
            "title": "总结标题(如: 未来展望)",
            "question": "一个引发思考的开放性问题"
        }
    ]
}

## 示例参考:
如果论文是关于图像识别的，好的标题应该是:
- ✅ "让手机也能'看懂'世界：新一代视觉识别技术"
- ✅ "比人眼还准？这个AI如何识别万物"
- ❌ "基于深度学习的图像识别算法研究" (太学术)
- ❌ "论文解读" (太空洞)

请确保所有内容都通俗易懂，避免学术术语。
```

## 2. 引言生成 Prompt (Writer - Intro)

**位置**: `paper_to_popsci/core/writer.py` - `_generate_intro()`

**作用**: 生成文章开头的引子部分

**Prompt 内容**:

```
请写一篇面向"一无所知"小白的科普引言。

论文标题: {paper_content.title}
核心创新: {outline_data.get('core_innovation', '')}
类比主题: {analogy_theme}
类比场景: {analogy}

要求:
1. 用生活化的故事场景引入，比如日常生活、吃饭、购物、交通等场景
2. 专业术语首次出现时，用*术语（大白话解释）*格式，例如：*神经网络（像人脑一样工作的计算程序）*
3. 禁止出现Markdown格式（如##、###、- 等）
4. 禁止出现任何数学公式或LaTeX符号（如$、^、_等）
5. 用设问引导读者思考
6. 结尾引出"这和论文主题有什么关系"
7. 200-300字，语言生动有趣，像跟朋友聊天
8. 使用第三人称客观叙述

直接输出正文内容，不要加标题。
```

## 3. 问题背景 Prompt (Writer - Problem)

**位置**: `paper_to_popsci/core/writer.py` - `_generate_problem()`

**作用**: 解释现有方法的问题和局限

**Prompt 内容**:

```
请用大白话向"完全不懂"的读者解释现有方法的问题。

论文标题: {paper_content.title}
核心创新: {outline_data.get('core_innovation', '')}
现有问题: {pain_point}
类比主题: {outline_data.get('analogy_theme', '日常生活')}

论文摘要: {paper_content.abstract[:500]}

要求:
1. 用生活化类比解释问题，比如做饭、搬家、整理房间等场景
2. 专业术语首次出现时，用*术语（大白话解释）*格式
3. 禁止出现Markdown格式（##、###、- 等）
4. 禁止出现任何数学公式或LaTeX符号（$等）
5. 用设问引出"那怎么办呢？"
6. 300-400字，像给爷爷奶奶讲解一样耐心
7. 使用第三人称客观叙述

直接输出正文内容。
```

## 4. 核心方法 Prompt (Writer - Method)

**位置**: `paper_to_popsci/core/writer.py` - `_generate_method()`

**作用**: 解释论文的核心技术方案

**Prompt 内容**:

```
你是一位擅长科普的技术作家。请用通俗易懂的语言向完全不懂技术的读者解释这篇论文的核心方法。

论文标题: {paper_content.title}
核心创新: {outline_data.get('core_innovation', '')}
关键概念: {concepts_text}
类比主题: {outline_data.get('analogy_theme', '日常生活')}

论文方法章节原文: {method_content}
论文摘要: {paper_content.abstract[:500]}

【重要】写作要求:
1. 必须深入解释具体的技术方案，不能只说"核心方法"、"关键技术"这种空话
2. 必须从论文原文中提取具体的方法细节，比如:
   - 具体的算法步骤或架构设计
   - 关键的技术组件或模块
   - 创新点的工作原理
3. 用生活化类比解释每个关键步骤，比如:
   - "就像做菜时先备料再炒菜"
   - "类似于搭积木，先打地基再往上搭"
4. 禁止使用"首先、其次、最后"这种套话结构
5. 禁止使用"核心方法"、"关键技术"、"优化算法"这种空洞词汇
6. 专业术语必须用*术语（通俗解释）*格式，例如: *注意力机制（让模型知道该重点关注哪些信息）*
7. 禁止出现Markdown格式（##、###、- 、**等）
8. 禁止出现任何数学公式或LaTeX符号
9. 400-600字，像给朋友讲解一样自然流畅
10. 必须让读者看完后能说出"原来是这么做的"

直接输出正文内容，不要标题。
```

**如果检测到空洞内容，会使用更严格的重试 Prompt**:

```
请重新写作，这次必须包含具体技术细节。

论文的方法部分说了什么？请从以下内容中提取关键信息:
{method_content}

要求:
- 必须说明具体做了什么（不能只说"提出了新方法"）
- 必须解释为什么这样做能解决问题
- 用生活化例子类比每个关键步骤
- 300-500字，通俗易懂

直接输出正文。
```

## 5. 实验结果 Prompt (Writer - Results)

**位置**: `paper_to_popsci/core/writer.py` - `_generate_results()`

**作用**: 总结实验结果和性能指标

**Prompt 内容**:

```
请用大白话向"小白"总结实验结果。

论文标题: {paper_content.title}
关键指标: {metrics_text}

论文结果章节: {paper_content.sections[3].content[:800]}

要求:
1. 用具体数字说话，并解释这些数字意味着什么
2. 专业术语首次出现时，用*术语（大白话解释）*格式
3. 用生活化对比（比如"比原来快了3倍，就像从走路变成坐汽车"）
4. 禁止出现Markdown格式（##、###、**等）
5. 禁止出现任何数学公式或LaTeX符号（$等）
6. 200-300字，让普通人也能理解这些数字的意义
7. 使用第三人称客观叙述

直接输出正文内容。
```

**作用**: 总结全文并展望未来

**Prompt 内容**:

```

2. 提出一个开放性问题引发思考
3. 用一句通俗易懂的"金句"结尾
4. 专业术语首次出现时，用*术语（大白话解释）*格式
5. 禁止出现Markdown格式（##、###、> 等）
6. 禁止出现任何数学公式或LaTeX符号（$等）
7. 150-200字，温暖、启发性
8. 使用第三人称客观叙述

直接输出正文内容。
```

## 常见问题

### Q1: 为什么生成的标题是"论文解读"？

**原因**: Analyzer 的 JSON 解析失败，或者 LLM 没有按要求生成吸引人的标题

**解决方案**:
1. 检查 Analyzer 的输出日志
2. 确认 LLM 返回了有效的 JSON
3. 查看 hero section 的 title 字段是否有内容

### Q2: 为什么内容很空洞，都是套话？

**原因**: 
1. LLM 生成失败，使用了默认模板（已移除）
2. API 速率限制导致内容不完整
3. Prompt 不够具体

**解决方案**:
1. 已优化所有 Prompt，增加了具体要求
2. 已添加空洞内容检测和重试机制
3. 确保 API 服务稳定

### Q3: 为什么有些段落文字不全？

**原因**:
1. API 返回被截断（max_tokens 限制）
2. 网络问题导致响应不完整
3. 速率限制导致请求失败

**解决方案**:
1. 已增加 max_tokens 限制
2. 已添加重试机制
3. 已添加请求间隔控制

## 优化建议

如果生成的内容质量不佳，可以考虑:

1. **调整 temperature**: 
   - 降低 temperature (0.5-0.6) 让输出更聚焦
   - 提高 temperature (0.8-0.9) 让输出更有创意

2. **增加上下文**:
   - 提供更多论文原文内容
   - 增加示例参考

3. **细化 Prompt**:
   - 添加更多具体要求
   - 提供正面和反面示例

4. **后处理**:
   - 检测空洞内容并重新生成
   - 验证输出格式和长度
